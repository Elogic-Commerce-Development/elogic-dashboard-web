# Impeccable Design Context — Elogic Dashboard

## Users
Internal analytics tool for the Elogic team (~90 employees). Used by:
- **Project managers** tracking estimate vs actual on individual tasks
- **Team leads** reviewing how their reports are spending time
- **Ops / leadership** spotting overrun tasks and capacity problems

Used at a desk on a laptop, almost always during a working session — not on
mobile, not on the go. Users come in to answer a specific question (is this
task overrun? did this employee track their week?) then leave. They will
look at the same view dozens of times per week, so legibility-at-a-glance
matters more than first-time delight.

## Brand Personality
**Operational. Dense. Honest.** Three words.

It's a tool for running a software services company, not a marketing
surface. It should feel like a well-built spreadsheet built by someone who
respects the reader's time — closer to Linear or Notion's data tables than
to a Stripe marketing page. Numbers should be tabular and easy to compare;
hierarchy should be clear without shouting; color should mean something
(red = something is wrong, amber = caution, emerald = good, blue =
informational, violet = creative work, neutral = nothing to flag).

Anti-references:
- No gradient text, no glassmorphism, no decorative side stripes on cards.
- No marketing-style hero sections, no oversized rounded icons above headings.
- No animation that makes the user wait. Period switching should feel instant.

## Aesthetic Direction
**Light theme.** Users sit in well-lit offices and home offices reading
dense data; light is right. Background tints toward neutral-50 with a
faint warm cast (matches the existing implementation).

Typography is system-driven — `font-family` already set to system stack;
keep it. Numbers are tabular (`tabular-nums`) so columns line up.

Existing colour palette (Tailwind):
- emerald-600 = good / tracked
- red-600 = bad / missing / overrun
- amber-600 = caution / sick
- blue-600 = informational / vacation
- violet-600 = creative work / design
- neutral-* = chrome and surfaces

Job-type colours are centralised in `src/lib/jobTypes.ts` and MUST be
honoured wherever job-type segments appear (consistent legend recognition
across the dashboard is part of the brand).

## Design Principles
1. **Numbers are the hero.** Charts illustrate; the actual digits are what
   the reader walks away with. Always show the underlying number.
2. **Color carries meaning, not decoration.** If it's not communicating
   status / category / scale, it shouldn't be coloured.
3. **Visual rhythm through density variation, not whitespace inflation.**
   The dashboard packs a lot of information per screen — that's a feature.
   Group tightly, separate intentionally.
4. **Text is left-aligned, numbers are right-aligned.** Tabular data only.
5. **Reuse before invention.** Existing patterns (KPI cards with semantic
   tone, segmented pills, dense tables) read consistently across pages.
   Don't introduce a new card style for a new section.
